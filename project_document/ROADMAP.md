# Roadmap

## 定位

`agent-aigc` 是面向创作者的 AIGC 创作平台。它继承 `infra-dev-scaffolding` 的工程规范，但业务重点不在通用脚手架，而在多模型调度、Prompt 优化链、文生图/音频/视频、创作历史和模型参数配置。

## V1 目标

1. 用户只输入一句话，系统自动识别创作类型、优化 Prompt、选择模型并创建任务。
2. 图片、视频、音频共用一套任务、资产、历史和 Provider 路由模型。
3. 本地开发默认无 Key 可演示；真实模型通过环境变量或后续配置页接入。
4. 作品能沉淀为资产，并可发布到灵感广场。
5. 工程上持续通过脚手架质量门禁，避免业务迭代破坏基础契约。

## 非目标

- V1 不做复杂工作流编排。
- V1 不做团队协作、计费、权限套餐和模型市场。
- V1 不把 mock provider 当真实生成能力。
- V1 不追求所有模型都生产可用，优先打深一条图片生成链路。

## 业务边界

| 模块 | 职责 |
| --- | --- |
| 创作工坊 | 一句话输入、参考素材、Agent 决策展示、生成进度、结果预览 |
| Agent 链路 | 意图识别、Prompt 清洗、Prompt 增强、模型选择 |
| Provider 路由 | 按内容类型和配置选择图片/视频/音频 provider |
| 任务中心 | 任务状态、进度、错误、耗时、生成结果 |
| 资产库 | 个人作品、下载、删除、发布到广场 |
| 灵感广场 | 已发布作品展示、类型/模型/关键词筛选 |
| 模型配置 | Provider 开关、默认模型、参数模板、可用性检查 |

## 阶段规划

### V0.8: 可演示 MVP

验收：
- 无 OneRouter Key 时使用规则意图识别。
- 无 Google Key 时使用 mock provider 完成图片/视频/音频任务。
- 任务异步执行，前端可轮询进度。
- 生成结果保存为资产。
- 广场支持基础筛选。
- `./scripts/check-contracts.sh`、`mvn -q test`、`./scripts/quality-gate.sh` 可通过。

### V0.9: 创作体验收口

验收：
- 创作输入支持显式类型选择和自动识别两种模式。已完成基础模式切换。
- Prompt 优化结果可解释：清洗后的 Prompt、增强项、参考素材策略。已完成清洗 Prompt、参数摘要和置信度展示。
- 历史记录支持重新生成、复制 Prompt、发布、删除。已完成 Prompt 复用、发布和删除。
- 视频/音频 mock 明确显示为本地预览，占位结果不误导用户。
- 模型列表从后端配置和 provider 可用性派生。已完成创作页能力条和只读模型配置页。

### V1.0: 真实图片生成链路

验收：
- Google 图片 provider 在真实 key 下可稳定生成并保存文件。
- 本地存储路径、URL 前缀、缩略图和清理策略可配置。已完成本地生成物保存服务收口，缩略图和清理策略待补。
- 失败结果有明确错误码、用户提示和重试入口。
- 图片参数支持宽高比、尺寸、风格和参考图。
- OpenAPI 类型生成和前端 API model 保持一致。

### V1.1: 素材库与参数配置

验收：
- 上传 API 支持图片素材，落本地或 OSS。
- 素材库支持预览、删除、引用到创作任务。
- 模型配置页支持 provider、模型、默认参数、可用性探测。
- 任务记录保留实际 provider、模型、参数、耗时和错误摘要。

### V1.2: 多模态增强

验收：
- 视频 provider 具备至少一条真实生成或异步回调链路。
- 音频 provider 具备 TTS 或音乐生成的一条真实链路。
- 作品广场支持类型 tabs、Prompt 复用、来源模型筛选。
- 成本、耗时、失败率开始沉淀为可观察字段。

## 当前优先级

1. 稳住 V0.8：无 Key 演示、异步任务、资产和广场筛选。
2. 推进 V0.9：前端体验、Prompt 解释、历史操作。
3. 选择一条真实图片 provider 做 V1.0 生产级闭环。
4. 保持脚手架契约和质量门禁，不让业务代码绕过统一 API、时间、上下文和 OpenAPI 约束。

## 工程契约

- 日志统一输出 requestId、traceId、用户、租户、路径、耗时和错误码。
- 异步任务继续由 `scripts/check-async-context-contract.js` 守护上下文传播。
